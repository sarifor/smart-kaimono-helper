## 1안: 키워드를 입력하면 장바구니 목록 생성

### 사용 흐름

- 터미널에 python shop.py 를 실행한다.<br/>
- "이번 주 예산이랑 먹고 싶은 거 입력해줘" 라는 메시지가 뜨면, 답을 입력한다. (예: 3000엔, 야채 위주에 달걀이랑 두부 꼭 넣어줘)<br/>
- Claude가 장바구니 목록을 생성하고,<br/>
- ✖ 브라우저가 자동으로 열리며 넷슈퍼에 로그인한다.<br/>
- ✖ 에이전트가 상품을 하나씩 검색하고 카트에 담는 걸 터미널 로그로 지켜본다.<br/>
- ✖ 상품을 검색할 수 없으면 패스한다.<br/>
- ✖ 다 끝나면 완료! 총 2,840엔 / 12개 상품 담았어요 가 출력된다.<br/>
- ✖ 담지 못한 상품 목록도 출력한다.<br/>
- ✖ 나머지는 사용자가 직접 결제<br/>
- 이후 단계(장바구니 담기, 결제 등)는 사용자 몫.


### 원리

- 사용자가 터미널에서 python shop.py 를 실행하면, Python이 .env 파일을 읽어온다.<br/>
- Python이 사용자 입력(한국어 or 일본어, 혹은 혼용 OK)을 받아 Claude API에 던진다.<br/>
- Claude는 웹 검색 없이 학습 지식 기반으로 "1인 가구 일본 슈퍼 기준, 완성품 위주에 기본 원재료 적절히 섞어서" 상품 목록을 JSON으로 뽑아낸다. (일본어)<br/>
- ✖ Python이 그 JSON을 파싱해, browser-use에 넘기면, browser-use가 브라우저를 열고 넷슈퍼에 로그인한 뒤 상품을 하나씩 검색한다.<br/>
- ✖ 검색 결과가 여러 개 뜨면 넷슈퍼 추천 순 최상단 상품을 선택하고, 누적 금액이 예산을 넘지 않는 선에서 카트에 담는다.<br/>
- ✖ 검색이 안 되거나 담으면 예산을 초과하는 상품은 스킵한다.<br/>
- ✖ 전부 끝나면 완료! 총 2,840엔 / 12개 상품 과 함께 담지 못한 목록을 터미널에 출력한다. 


### 기타

#### 기본 정보

- 프로젝트명: smart-kaimono-helper<br/>
- 메인 파일: shop.py<br/>
- 스택: Python + Claude API + ✖ browser-use<br/>
- 대상 사이트: 넷슈퍼


#### 환경

- Anthropic API 키: console.anthropic.com 에서 가입 후 API Keys → Create Key. 발급된 키를 .env 에 붙여넣기.<br/>
- .env 파일 세팅: Anthropic API 키 써 두기<br/>
- Python 환경 + 라이브러리 설치: pip install ✖browser-use anthropic python-dotenv (전역 설치)<br/>
- ✖ 4. 넷슈퍼 웹 브라우저 사전 로그인: shop.py 실행 전에 웹 브라우저에서 넷슈퍼에 직접 로그인해두기.

#### 참고

- Claude Sonnet 사용: 장바구니 목록 생성은 애매한 한국어·일본어 입력을 일본 슈퍼 상품명으로 구체화하는 추론이 필요하기 때문이다. extended thinking(auto/medium/high)은 켜지 않는다. 수학 증명처럼 복잡한 추론이 아니라 Sonnet 기본(thinking 파라미터 없음)으로 충분하다.

- Claude Sonnet API 비용: input $3/1M 토큰, output $15/1M 토큰이다. 이 앱에서 한 번 실행할 때 시스템 프롬프트와 사용자 입력을 합쳐 약 500 토큰, 출력 JSON 목록이 약 200 토큰으로, 실행 한 번에 0.5엔 이하다. 일주일에 10번 써도 약 5엔, 한 달이면 20엔 수준이라 사실상 무시해도 되는 금액이다. 첫 가입 시 무료 크레딧도 제공되므로 당분간은 비용 없이 테스트할 수 있다.

- Claude API 호출 시 browser-use 내장 ChatAnthropic 대신 anthropic SDK를 직접 사용: ChatAnthropic 은 browser-use가 중간에 끼어 Claude API를 대신 호출하는 구조라 라이브러리 업데이트에 따라 동작이 바뀔 수 있기 때문이다. anthropic SDK 직접 호출은 중간 레이어 없이 Anthropic 공식 SDK를 그대로 쓰는 거라 더 안정적이고 예측 가능하다.

- ✖ 사용 LLM: 장바구니 목록 생성(Step 1)과 브라우저 조작(Step 2) 모두 Claude Sonnet 4.6을 사용한다. Sonnet은 Opus 대비 비용이 저렴하면서도 한국어·일본어 혼용 입력을 일본 상품명으로 변환하는 추론과 브라우저 화면을 보고 클릭·검색하는 판단 모두 충분히 소화할 수 있으며, browser-use 공식 문서의 Anthropic 지원 모델로 명시되어 있어 연동 안정성도 보장된다. Haiku는 빠르고 저렴하지만 브라우저 조작처럼 화면을 보고 맥락을 파악해야 하는 복잡한 판단에는 약할 수 있어 선택하지 않았다. 주의할 점은 Step 2의 브라우저 조작이 화면을 반복적으로 읽고 판단하기 때문에 Step 1보다 토큰 소비가 훨씬 많아 실행 비용의 대부분을 차지한다는 것이다.

- ✖ Python은 Claude → browser-use → 결과 출력까지 전체 흐름을 잇는 접착제 역할

- ✖ Python을 쓰는 이유는 단순한데, browser-use가 Python 라이브러리이기 때문이다. TypeScript 등 다른 언어로는 같은 걸 하려면 처음부터 직접 다 짜야 해서 공수가 몇 배로 늘어난다.

- ✖ browser-use가 넷슈퍼 웹사이트를 직접 보기 때문에 별도의 외부 검색 없이도 실제 재고와 가격을 정확하게 확인할 수 있다.
